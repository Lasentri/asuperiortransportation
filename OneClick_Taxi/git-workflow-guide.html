🐙 Step 1 — GitHub Repository Setup

Create a new repository on GitHub.com

Go to github.com/new and fill in:

  • Repository name: e.g. my-taxi-site
  • Visibility: Private (recommended — keeps your code secure)
  • Initialize with README: ✅ Check this

Click Create repository.

Generate a GitHub Personal Access Token

Your server needs this to pull code from GitHub securely.

  • Go to GitHub → Your Profile → Settings
  • Scroll to Developer settings → Personal access tokens → Tokens (classic)
  • Click Generate new token (classic)
  • Name it: my-server-deploy
  • Expiration: No expiration (or 1 year)
  • Scopes: Check repo (full repository access)
  • Click Generate token
  • ⚠️ COPY IT NOW — you won't see it again
⚠️
Save your token somewhere safe like a password manager. It looks like: ghp_aBcDeFgHiJkLmNoPqRsTuVwXyZ123456

Install GitHub Desktop and clone your repo

  • Download and install GitHub Desktop
  • Sign in with your GitHub account
  • Click File → Clone Repository
  • Select your repo from the list
  • Choose a local folder — e.g. C:\Sites\my-taxi-site
  • Click Clone

This creates a local copy of your repository on your computer.

Copy your WordPress plugin files into the repo folder

Navigate to your cloned repo folder and copy your plugin files there. For example, copy your oneclick-taxi plugin folder into the repo.

GitHub Desktop will automatically detect the new files and show them as "changed".

Make your first commit and push

  • In GitHub Desktop, you'll see all your files listed under "Changes"
  • In the bottom left, type a summary: Initial commit - OneClick Taxi plugin
  • Click Commit to main
  • Click Push origin (top right)

Your files are now on GitHub. ✅

🖥️ Step 2 — Server Setup

Connect to your server with PuTTY (Windows) or Terminal (Mac) and run these commands.

Navigate to your WordPress site directory

BASH
cd /var/www/yoursite.com

Replace yoursite.com with your actual site folder name.

Initialize Git and connect to your GitHub repo

BASH
git init
git remote add origin https://YOUR_TOKEN@github.com/YOUR_USERNAME/YOUR_REPO.git
git fetch origin
git checkout -b main origin/main

Replace YOUR_TOKEN, YOUR_USERNAME, and YOUR_REPO with your actual values.

💡
Example: https://ghp_abc123@github.com/JohnDoe/my-taxi-site.git

Test that git pull works

BASH
git pull origin main

You should see: Already up to date. or a list of files pulled. If you get an authentication error, double-check your token.

Set correct file permissions

BASH
chown -R www-data:www-data /var/www/yoursite.com
find /var/www/yoursite.com -type f -name "*.php" -exec chmod 644 {} \;
find /var/www/yoursite.com -type d -exec chmod 755 {} \;

🔔 Step 3 — Auto-Deploy Webhook

This is the magic. Every time you push to GitHub, your server automatically pulls the new code. No manual steps.

Create the deploy script on your server

BASH
nano /var/www/yoursite.com/deploy.php

Paste this code into the file:

PHP
<?php
/* Auto-deploy webhook — triggered by GitHub push */
$secret = 'YOUR_WEBHOOK_SECRET'; /* Change this to something random */
$logfile = '/var/log/deploy.log';

/* Verify GitHub signature */
$payload   = file_get_contents('php://input');
$signature = 'sha256=' . hash_hmac('sha256', $payload, $secret);
$received  = $_SERVER['HTTP_X_HUB_SIGNATURE_256'] ?? '';

if ( ! hash_equals($signature, $received) ) {
    http_response_code(403);
    file_put_contents($logfile, date('Y-m-d H:i:s')." UNAUTHORIZED\n", FILE_APPEND);
    exit('Unauthorized');
}

/* Pull latest code */
$output = shell_exec('cd /var/www/yoursite.com && git pull origin main 2>&1');
file_put_contents($logfile, date('Y-m-d H:i:s')." DEPLOY:\n".$output."\n", FILE_APPEND);

http_response_code(200);
echo 'Deployed: ' . date('Y-m-d H:i:s');
?>

Save with Ctrl+X → Y → Enter. Replace YOUR_WEBHOOK_SECRET with a random string like mySecretKey2026!

Create the deploy log file

BASH
touch /var/log/deploy.log && chmod 666 /var/log/deploy.log

Add the webhook on GitHub

  • Go to your GitHub repository
  • Click Settings → Webhooks → Add webhook
  • Payload URL: https://yoursite.com/deploy.php
  • Content type: application/json
  • Secret: the same secret you put in deploy.php
  • Events: Just the push event
  • Click Add webhook
💡
After saving, GitHub will send a test ping. Click on the webhook and scroll to "Recent Deliveries" — you should see a green ✅ for the test ping.

Test the full pipeline

Make a small change — add a comment to any file in GitHub Desktop, commit it, push it. Then on your server run:

BASH
tail -f /var/log/deploy.log

You should see the deploy log update automatically within seconds of pushing. That's it — your pipeline is live. 🎉

Once set up, every development session follows this loop:

  1. Tell Claude what you want to build or fix
  2. Claude writes the code and pushes it to GitHub
  3. You run git pull origin main on the server (or the webhook does it automatically)
  4. Refresh your browser — change is live
  5. If something breaks, paste the error into Claude — it fixes it
  6. Repeat

When git pull fails — the fix

Sometimes your server has local changes that conflict with what's on GitHub. Always fix this with:

BASH
git checkout -- wp-content/plugins/your-plugin/filename.php
git pull origin main

Or to discard ALL local changes and force-pull:

BASH — NUCLEAR OPTION
git fetch origin
git reset --hard origin/main
⚠️
The nuclear option discards ALL local changes. Only use it if you're sure everything important is already on GitHub.

Cache busting — force browser to load new files

When you update CSS or JS files, WordPress may serve the old cached version. Bump the version number to bust the cache:

BASH
sed -i "s/define( 'OCT_VERSION', '1.0.0'/define( 'OCT_VERSION', '1.0.1'/" \
  wp-content/plugins/oneclick-taxi/oneclick-taxi.php

Or ask Claude to bump the version — it'll update the main plugin file and push it automatically.

🔧 Troubleshooting

"Permission denied" on git pull

Your token may have expired or be incorrect. Generate a new token on GitHub and update the remote URL:

git remote set-url origin https://NEW_TOKEN@github.com/USERNAME/REPO.git

Webhook not firing

Check GitHub → Settings → Webhooks → your webhook → Recent Deliveries. If you see a red ✗, click it and read the response. Common causes:

  • Your deploy.php URL is wrong
  • The secret doesn't match
  • PHP is not allowed to run shell_exec

Test shell_exec on your server:

php -r "echo shell_exec('echo hello');"

Site shows old version after pushing

Either the webhook didn't fire, or browser cache. Try:

# Force pull manually
cd /var/www/yoursite.com && git pull origin main

# Hard refresh browser
Ctrl+Shift+R (Windows/Linux)
Cmd+Shift+R (Mac)

Claude pushed code but I can't see it on my server

Run git pull manually. Sometimes the webhook fires but the pull fails due to local changes. Check the deploy log:

cat /var/log/deploy.log | tail -20

If it shows merge conflicts, paste the output into Claude and it will tell you exactly what to run.

Everything broke and I don't know why

Open Claude Pro, paste the error message, and say: "This broke after the last push. Here is the error. Here is the deploy log. Fix it."
